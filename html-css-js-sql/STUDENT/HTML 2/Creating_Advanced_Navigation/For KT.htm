Lesson 1
Creating Advanced Navigation

Lesson Objectives:

In this lesson, you will create advanced navigation.

You will:

  • Create CSS-based menus.
  • Incorporate images in CSS-based menus.
  • Create image maps.

Introduction

You have tested the validity of markups used in your website. However, a valid website serves no purpose if a user cannot easily browse through its pages. In this lesson, you will create advanced navigation.

Links are the essence of the World Wide Web. They enable a web page to connect to multiple locations on the Internet. Websites support this principle of interconnectivity to provide users with an easy means of accessing different web pages within or outside a website. Absence of links may lead users to lose their trail in your website, being stuck on a dead-end web page, or having no clue on how to get back to the home page. Such negative experiences may discourage users from visiting your website in the future.


Topic A
Create CSS-Based Menus

You familiarized yourself with the testing of markups in a CSS file. A means of navigation is very important for users to access different web pages in your website. However, you might wish to provide users with more structured navigation options that are clearly visible. In this topic, you will create CSS-based menus.

Users often waste a great deal of time trying to navigate to the content they are interested in, or simply trying to find their way around a website. Hypertext links are not particularly noticeable and may be overlooked, especially if the website contains a lot of textual content. Moreover, the links are not particularly feasible when a website contains many pages. Providing a clear-cut, well-positioned, and visually attractive navigation options is a professional practice that avoids such problems.


You might wish to provide users with more structured navigation options that are clearly visible. In this topic, you will create CSS-based menus.

Common Menu Types

Menus are a common method of providing users with navigation options to browse through the various pages of a website. CSS-based menus comprise hypertext links that are organized into lists with applied formatting defined in the CSS. Orientation of the menu, either vertically or horizontally, can also be specified using CSS. These menus usually contain items that reflect the structure of the website. Menu items can also include submenus based on the website's structure.




Figure 1-1:


A CSS based menu may have multiple levels of submenus. However, using too many sublevels for a menu is not a good practice because it hinders the ease of navigation for users.

The Display Property

The display property is used to control the appearance of an element on a web page.

Display Property Value Description
display: block Displays an element with a line break above and below it.
display: inline Displays an element without any line breaks above or below it.
display: none The element is not visible on the web page. The browser does not display blank space in the element's place.
display: list-item Displays an element as a list. This style rule is not supported by IE 6 or earlier versions.

The :hover Pseudoclass

    You can create rollovers for various elements, other than links, on a web page using the :hover pseudoclass. Internet Explorer 7 and earlier versions can only apply the :hover pseudoclass to the links. So, the menus that use this class may not appear correctly. In such cases, you can add a script code to support rollovers for these elements.

How to Create CSS-Based Menus

Procedure Reference: Create CSS-Based Menus

To create CSS-based menus:

  1. In a text editor, open the web page for which you want to add a menu.
  2. In the header section of the markup, within the nav element, create an unordered list of menu items using the <ul> element.
  3. Within the <ul> element, create list items using the <li> elements.
  4. Within each list item, create a hypertext link to another page.
  5. Define styles for the normal and hover effect for the menu in the stylesheet.
  6. Save the file.
  7. Launch the web page in a browser and move the mouse over the CSS-based menu to view the mouse-over effect.
  8. If necessary, copy the unordered list and paste it in other web pages of the website to maintain consistency in the visual appearance of navigation links.

Procedure Reference: Create Popup Submenus

To create popup submenus:

  1. In a text editor, open the web page for which you need to add a submenu.
  2. For each menu that requires a submenu, create an unordered list of submenu items.
  3. Within the submenu unordered list, add the required hyperlinks for the relevant content pages.
  4. Save the file.
  5. If necessary, add this code to other web pages in the website to maintain consistency.

By using images in the menu, you achieve this effect without overshadowing the website's content.


It is important to make the menus visually pleasing in order to attract the users' attention. In this topic, you will incorporate images in CSS-based menus.

Image Sprite

An image sprite is a single image that is formed by a collection of images. Often, a web page contains a lot of images. Loading these images individually generates multiple server requests resulting in an increased bandwidth usage and a longer time to load the web page. Image sprites allow you to reduce the number of server requests and save the available bandwidth.

Navigation List with Image Sprites

Image sprites allow you to display individual sections of a composite image, such as a navigation list. Image sprites allow you to save the bandwidth because only one image is downloaded on the user's computer instead of multiple images for different states of each button. The performance of the web page also improves, because the entire image is downloaded when the page loads, instead of each image downloading as and when required based on user interaction.

Topic C
Create Image Maps

You have incorporated images in CSS-based menus to make them look attractive. In certain circumstances, you may need to use a purely image-based navigation system rather than menus or hypertext links. In this topic, you will create image maps.

Images are a powerful means of interacting with users. The use of graphical navigation options provides users with a more intuitive and enjoyable way of navigating to specific content within your website. In addition, this feature will help you make your website stand apart from other sites on the web.


You may need to use a purely image-based navigation system rather than menus or hypertext links. In this topic, you will create image maps.

Image Maps

An image map is a type of image that contains predefined links. You can access these links on the image to view any additional information. The image for the image map is added using the <img> tag, and the links are specified using the usemap attribute. HTML allows you to create two types of image maps, the client-side image maps and server-side image maps.

Image map Description
Client-side image maps Are defined by using the <map> tag. You can define the coordinates for the linked areas on the image. The web servers do not process the data to calculate the location of mouse over the image.
Server-side image maps Are similar to the client-side image map with one exception. When a user clicks on any part of the map, the horizontal and vertical coordinates of the mouse pointer cursor are sent to the server. The server processes this information and responds with an appropriate result that will be displayed on the browser.




Figure 1-3:


You cannot define an image map using the usemap, if the entire image is defined as a hyperlink using the <a> tag.

Hotspot Coordinates

    Creating an image map requires the coordinates of its hotspots. You can get these coordinates by opening the image in a graphics editing software, such as Microsoft Paint, and placing the mouse pointer over a particular point. The coordinates of this point will be displayed on the status bar at the bottom of the window. You can note down the required coordinates and use them to define the hotspots.

The <map> Tag

The <map> tag is used to define a client-side image map. The main advantage of this <map > tag is that it is supported by all major browsers such as Internet Explorer, Opera, Firefox, Safari, and Chrome.

The <area> Tag

The <area> tag is used to define a hotspot area within an image map. The <area> tag provides different attributes that enable you to define a hotspot of specific shape to suit your needs.

Attribute Description
coords Defined in coordinates and it specifies the coordinates within an area.
href Defined as a URL and it specifies the hyperlink's target.
nohref When set, implies that the area does not have an associated link.
shape Specified as default, rect, circle, or poly to indicate the shape of an area in a map.
target Sets the target location where the linked file will be opened. The four values that it accepts include _blank, _parent, _self, and _top.

How to Create Image Maps

Procedure Reference: Create an Image Map

To create an image map:

  1. In the text editor, open the web page for which you want to add an image map.
  2. Add the image to be used as the image map using the <img> tag.
  3. In the <img> element, to define the coordinates of the mapped areas in images, set the value of the usemap attribute
  4. Below the <img> tag, define the map element.
  5. Enter the map attributes to define the hotspot areas on the map.
  6. Save the file.
  7. If necessary, view the web page in a browser and click a hotspot to load the corresponding page.
