You may want to provide the users with metadata of your website to make it more easily attainable to the users. In this topic, you will describe tags used for defining metacontent.

The <meta> Tag

The <meta> tag is usually placed in the <head> section of an XHTML document. It provides the metadata information about the HTML document. It is an empty tag, but includes various attributes that allow you to describe the web page content.

Attributes of the <meta> Tag

The <meta> tag includes various attributes.

Attribute Used To
charset Specify the character encoding of the HTML documents. Its value is given in character_set.
content Give the value of the http-equiv or the name attribute. Its value is given in text.
http-equiv Provide HTTP header. Its value is given in content-type, default-style, and refresh.
name Specify a name for metadata. Its value is given in application-name, author, description, generator, and keywords.

The http-equiv Attribute

The http-equiv attribute is used to provide a HTTP header for the value or information of the content attribute. It can be used to simulate a HTTP response header. The content attribute gives the value associated with the http-equiv attribute. This attribute has different values.

Value Used To
content-type Specify the character encoding of the documents.
default-type Specify the correct style sheet to be used.
refresh Define a time interval for the document to refresh itself.

The Name Attribute

The name attribute is used to specify the name for metadata. It allows you to set a name for a value or information of a content variable.

Value Used To
application-name Specify the name of web application that is represented by the page.
author Specify the author name of the document.
description Specify the description of the page.
generator Specify the software package that is used to generate the document.
keywords Specify a list of keywords separated by comma that is relevant to the page.

Topic B
Embed Media Using Plug-Ins

You entered metadata for web pages to enable users to easily search for your web pages. In addition, you may want to ensure that the web page content looks rich and appealing. In this topic, you will embed media using plug-ins.

Rich interactivity is a key factor in drawing the interest of visitors to the website. By embedding media, you can add features to your website in terms of interactivity. A highly interactive website can provide users with an enjoyable experience.


You may want to ensure that the web page content looks rich and appealing. In this topic, you will embed media using plug-ins.

Plug-Ins

A plug-in is a type of software that helps you to add a specific capability to another software. They are often used in web browsers to display files of newer file types, play audio and video, and scan for viruses. Some commonly used plug-ins include Apple QuickTime and Adobe Flash Player.

Most browsers require plug-ins to play media files or display content embedded in web pages. The type of plug-in that is required depends on the format of the embedded file and the web browser. Different browsers may use different plug-ins to play media files of the same format. So, the software developers include specific plug-ins for each type of browser. For example, Adobe provides a Flash Player plug-in for Internet Explorer and another plug-in by the same name for Firefox.




Figure 1-1:

Plug-Ins Installation

    When installing a software, the plug-ins that are necessary to view the files of specific type for the web browsers are also installed. For example, installing Apple's QuickTime Player on your computer automatically installs the plug-ins for your browsers to play .mov files.


    Some plug-ins are provided by default; for instance, most systems that have Internet Explorer also includes a version of the Flash Player plug-in.

Streaming

When a web page contains an embedded media file, the file plays only after it is downloaded to the user's computer. When media files are available on a server, they can be streamed to the user's machine instead of downloading them. Streamed media files start playing directly on the browser as they are being downloaded. The advantage of streaming is that the media files can start playing without waiting for the entire files to be downloaded and the files are not saved locally on a user's machine. However, the user's Internet connection must have sufficient bandwidth to support streaming. Disadvantages of streaming include the cost of server software, quality of audio, and increased buffering times when playing media due to low speed Internet connection.

Buffering

    Buffering refers to the process of downloading data in chunks from an Internet source before it is played on the user's computer. Buffering duration depends on the speed of the user's Internet connection. If the speed is considerably high, then the buffering is hardly noticeable.

Media MIME Types

Media Multipurpose Internet Mail Extension (Media MIME) types identify the file format of media files embedded in web pages. This information helps browsers identify the appropriate plug-in to play the embedded media file. A MIME type comprises two parts—a type and a subtype—separated by a forward slash. A type is a generic categorization of the file such as an audio or video. A subtype is a specific definition of the type of file and is used to determine the type of application that will play files of that type. Examples of subtype include mpeg and mp4.

MIME Type Description
audio/x-ms-wma The MIME type for audio files with the extension .wma.
audio/x-wav The MIME type for audio files with the extension .wav.
audio/mpeg The MIME type for audio files such as MP3 files and Motion Picture Experts Group (MPEG) audio with the file extension .mpeg.
video/x-ms-wmv The MIME type for video files with the extension .wmv.
video/quicktime The MIME type for video files with the extension .mov.
application/x-shockwave-flash The MIME type for files with the extension .swf.
application/x-mplayer2 The MIME type to play files in Windows Media Player. It can be used for audio and video files.

Use of MIME Types

    MIME types may also include optional parameters. MIME types with the type application can be used to associate a file with a particular software application, making all browsers use the same application to play that file. MIME types are also referred to as Internet media types. They are not mandatory; their necessity varies depending on the browser and the media file's format.

The <embed> Tag

The <embed> tag is used to define a container for interactive content or external application that will be displayed using a plug-in. The <embed> tag is supported by all major browsers and provides several attributes.

Attribute Used To
height Set the height of the embedded file.. Its value is given in pixels.
src Specify the address of the external file that is to be embedded. Its value is given in URL.
type Specify the embedded contents MIME type. Its value is given in MIME_type.
width Set the width of the embedded content. Its value is given in pixels.

Topic C
Add Audio and Video

You embedded media in a web page using plug-ins. You may now want to add audio and video to play within the web page instead of opening it in a player on the user's computer. In this topic, you will add audio and video.

Incorporation of media is a key factor in drawing visitors to a website. HTML5 provides options to add media directly in a web page. You can use these features to enhance the user experience of your website with rich multimedia, without using any external software.


You may now want to add audio and video to play within the web page instead of opening it in a player on user's computer. In this topic, you will add audio and video.

Media File Formats

The term multimedia encompasses different types of media such as images, audio, video, and animation. Various file formats correspond to these different types of media. Certain file formats are proprietary to particular players and cannot be played in other players.

Media File Format Description
.wma This file format corresponds to audio files that play in most of the players like Windows Media Player, ALLPlayer, VLC media player, Media Player Classic, MPlayer, RealPlayer, and Winamp.
.wmv This file format corresponds to video files that play in Windows Media Player, RealPlayer, MPlayer, The KMPlayer, Media Player Classic, and VLC Media Player.
.mov This file format corresponds to video files that play in Apple's QuickTime Player.
.swf This file format corresponds to animations created with Adobe's Flash software.
.wav This file format corresponds to audio files.
.mp3 This file format corresponds to audio files.
.mpg and .mpeg These file formats corresponds to video files.

The <audio> Tag

The <audio> tag is used to define music, sound, or other audio streams to play in a web page. It provides various attributes that you can use to define how audio will be played on the web page. The <audio> tag is supported by all major browsers such as Internet Explorer, Firefox, Opera, Chrome, and Safari.




Figure 1-2:

Attributes of the <audio> Tag

The <audio> tag provides various attributes.

Attribute Used To
controls Specify the controls such as play and pause to be displayed on a web page. Its value is given in controls.
preload Define how the web designer wants the audio to load when the web page is accessed.
autoplay Specify that audio will play once it is ready. Its value is given in autoplay.
src Specify the URL of the audio files to be played on the web page.
loop Define whether audio will repeat after it is finished playing.

The <video> Tag

The <video> tag is used to specify video like video streams or a movie clip. It helps in enhancing the webpage by giving the provision to add video to the webpage. The <video> tag is supported by all major browsers such as Internet Explorer, Firefox, Opera, Chrome, and Safari.




Figure 1-3:

Attributes of the <video> Tag

The <video> tag contains various attributes.

Attribute Used To
autoplay Specify that video will play once it is ready. Its value is given in autoplay.
preload Specify how the author wants the video to load when the page is loading.
loop Specify that video will keep playing once it is finished. Its value is given in the loop.
src Specify the video files URL. Its value is given in URL.
controls Define the controls such as play and pause that will be displayed on the web page.
width Set the width of the video player. Its value is given in pixels.
muted Mute the audio of a video file when played on a web page.
height Set the height of the video player. Its value is given in pixels.
poster Specify the image that will be displayed when the video is downloading from a server. Its value is given in URL.

The <source> Tag

The <source> tag is used to incorporate multiple media resources for elements such as the <audio> and <video> tags. It enables you to specify multiple audio and video files, so that the browser can select the correct file based on the codecs or the media type it supports. This tag is supported by all major browsers. The <source> tag provides various attributes.

Attribute Used To
src Specify the media files URL. Its value is given in URL.
type Specify the type of media resource. Its value is given in MIME_type.
media Define the type of the media resource. Its value is given in media_query.
